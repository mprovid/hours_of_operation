Hours of Operation (Demo)

Purpose

This application is designed to provide multiple views of an organization's hours of operation. This application is to help make the exceptions to the regular hours easier for the user to understand and simpler for the web editor to maintain. Does your organization close or reduce hours on some holidays? Extend hours on some occasions? From a single JSON data set this application produces a list of regular hours and a list of exceptions by category. There is no set limit for how far in advance the regular hours and exceptions can be set. This application also produces calendar tables for months in a specified date range with the regular hours provided in the calendar header and exceptions indicated visually and as text in the calendar table cells. In the case of seasonal hours, months split where the season splits. Regular hours need to be in effect at least two (2) months since, as designed, a month can only split once -- any shorter span of time is an exception to regular hours.

At this stage of development, the javascript is not efficient and the CSS is written strictly for debugging purposes (for example, past calendar tables and exceptions in a list should be deleted, not grayed-out, and the current grayed-out style does not have adequate contrast for accessibility). This is a work in progress (both as proof of concept and prototype). Your collaboration is welcome.
